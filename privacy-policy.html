GlyphON Privacy Policy

Last updated: June 28, 2026

Introduction

GlyphON is an Android training overlay app. It helps users practise glyph sequences by showing an on-screen overlay, capture area, replay controls, preview area, and related configuration controls.

This Privacy Policy explains what information GlyphON collects, how permissions are used, where data is stored, and what choices users have. GlyphON is an independent application and is not affiliated with, endorsed by, or authorized by any third-party app or game.

Information collected

GlyphON does not collect personal information, account information, contact details, location data, photos, media files, microphone audio, camera data, advertising identifiers, analytics events, or crash reports.

The app processes user touch input locally while the overlay is active so it can capture and replay glyph strokes inside the training area. This input is used only for the app's training functionality on the device.

Permissions used

GlyphON currently declares the following Android permission:

  • SYSTEM_ALERT_WINDOW — allows GlyphON to display its training overlay above other apps after the user grants the Android “Display over other apps” permission.

GlyphON does not currently declare internet, location, camera, microphone, contacts, storage, notification, accessibility, or foreground service permissions.

SYSTEM_ALERT_WINDOW

GlyphON uses the SYSTEM_ALERT_WINDOW permission to show floating controls and a drawing/capture overlay on top of the current screen. This is essential to the app's purpose: the user needs to see and interact with the GlyphON training interface while another screen is visible underneath.

The overlay permission is used to create and manage visible overlay elements such as the capture bounds, preview area, play/reset/minimize/close controls, configuration controls, and replay display. GlyphON does not use this permission to read private content from other apps, collect personal data, or transmit information elsewhere.

Foreground Service

The current version of GlyphON includes an overlay service for managing the on-screen overlay, but it does not declare Android foreground service permissions in the application manifest. If this changes in a future release, this policy should be updated to describe the foreground service behavior and any related notification.

Data storage

GlyphON stores app preferences locally on the user's device using Android shared preferences. These settings may include options such as the selected color theme, glyph count, overlay scale, overlay opacity, auto-capture setting, floating control position, tutorial state, and whether glyph display is enabled.

This data remains on the device as part of the app's local configuration. GlyphON does not upload this information to a server.

Third-party services

GlyphON does not currently integrate third-party analytics, advertising networks, crash reporting services, cloud storage, or external data collection services.

User rights

Because GlyphON does not collect or transmit personal information, there is no server-side account data to access, export, correct, or delete. Users can control local app data by changing settings inside the app, clearing the app's data in Android system settings, or uninstalling the app.

Users can revoke the “Display over other apps” permission at any time in Android system settings. Revoking this permission will prevent the overlay functionality from working.

Contact

For privacy questions about GlyphON, contact the developer through the Google Play listing or the repository owner profile associated with the published project.
